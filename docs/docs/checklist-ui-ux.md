# Checklist de Verificação: UI & UX Design

Este checklist foi elaborado com base no **Guia de Boas Práticas para Acessibilidade Digital (Diniz et al., 2023)**, focado na interseção entre Design Universal e WCAG.

## 1. Design Visual e Interface (UI)
*O foco aqui é garantir que todos percebam os elementos da tela.*

- [ ] **Contraste de Texto:** O texto normal possui contraste mínimo de 4.5:1 e textos grandes 3:1.
- [ ] **Elementos de Interface:** Ícones e bordas possuem contraste mínimo de 3:1.
- [ ] **Uso da Cor:** A cor não é usada como única forma de transmitir informação (ex: erros têm ícones + texto).
- [ ] **Tipografia:** Fontes são legíveis (sans-serif) e redimensionáveis até 200% sem quebrar.
- [ ] **Foco Visível:** Elementos interativos mostram borda/indicador claro ao serem selecionados via teclado.

## 2. Conteúdo e Estrutura
- [ ] **Linguagem Simples:** Texto direto, sem jargões, frases curtas.
- [ ] **Cabeçalhos:** Uso lógico de H1, H2, H3 (apenas um H1 por página).
- [ ] **Texto Alternativo:** Imagens informativas têm descrição (`alt`), decorativas estão ocultas.
- [ ] **Links Descritivos:** O texto do link diz para onde ele vai (evite "Clique aqui").

## 3. Interação e Navegação (IHC)
- [ ] **Teclado:** Navegação completa possível apenas com Tab/Enter/Setas.
- [ ] **Ordem de Foco:** A navegação segue a ordem visual lógica.
- [ ] **Formulários:** Todos os campos têm `labels` e mensagens de erro claras.
- [ ] **Área de Toque:** Botões mobile têm pelo menos 44x44px (ou equivalente seguro).

## 4. Referência Bibliográfica
